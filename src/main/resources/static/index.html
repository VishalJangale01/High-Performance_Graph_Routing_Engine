The Problem (The "Sorting Barrier"): Standard routing algorithms like Dijkstra use a Binary Heap to manage routes. This forces the computer to "perfectly sort" every single node by distance, costing time: O(m log n).

The Solution (This Engine): This project implements the breakthrough 2025 research by Duan et al. Instead of perfect sorting, I engineered a Bucket Queue architecture. It groups nodes into "distance buckets" (rough sorting), allowing for near-linear time complexity: O(m).
